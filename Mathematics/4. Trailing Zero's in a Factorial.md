# Trailing 0 in a Factorial

Example:

1. I/P: 5
   O/P: 1 // Since factorial of 5 is 120
2. I/P: 10
   O/P: 2
3. I/P: 100
   O/P: 24

### Naive Solution

We first compute the factorial of the number and then check trailing zero in the factorial

```java

int naiveCountTrailingZero(int n) {
    int factorial = 1;
    for (int i = 2; i <= n; i++) {
        factorial = factorial * i;
    }

    int res = 0;
    while (factorial % 10 == 0) {
        res++;
        factorial = factorial / 10;
    }
    return res;
}
```

Time Complexity: θ(n)

But the major issue for this solution is it will cause overflow for even slightly higher value of n, i.e. factorial
cannot be stored in an int data type

### Efficient Method

If we are able to find the number of pairs of 5,2 as factors in the multiplication before the number n+1 then we can
determine number of zero in trailing.

For example: n = 11, we have 2,5 and then for 10 also it is 2,5. So in total 2 pairs so it will have 2 trailing zero.
When n = 15, Same 2 pairs are already present extra one 5 we will get from 15 and then 2 we can take out from any even
number. So we will have 3 trailing zero in this case.

Now we can notice that since 2 comes in every even digit as a factor we just have to count number of 5 factors that will
come as those would always be less than 2 factors.

If we have n = 17, then we can simply do 17/5 = 3, so three factors of 5 comes. But what if n = 25, then 25/5 = 5 but in
reality there are 6 factors of 5.

1. 5 -> 1
2. 10 -> 1
3. 15 -> 1
4. 20 -> 1
5. 25 -> 2

We can see that square, cube ... of 5 will start to give extra number of factors of 5. Hence, we need to divide n by
these number's as well: 5, 25, 125, 625 etc.


```java

int countTrailingZero(int n) {
    int res = 0;
    for(int i =5; i<=n; i = i*5) {
        res = res+ n/i;
    }
    return res;
}
```

Time Complexity: If this runs k times then,
                 5<sup>k</sup> <= n
                 k <= log<sub>5</sub>n

So we can say time complexity will be θ(log n)
