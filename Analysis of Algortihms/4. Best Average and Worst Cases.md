# Best, Average & Worst Cases

Let's use a function to understand this:

```java
/***
 *
 * @param arr is an array
 * @param n is size of the array
 * @return sum of the array if n is odd otherwise 0
 */
int getSum(int[] arr, int n) {
    if (n % 2 == 0) {
        return 0;
    }
    int sum = 0;
    for (int i = 0; i < n; i++) {
        sum += arr[i];
    }

    return sum;
}
```

`Best Case:` In best case we would get n as even hence we don't have to do any processing and just return 0. So Order of
Growth would be constant in this case.

`Average Case:` In average case we have to make assumption of what inputs can we get, for the above let us say we get
equal number of odd values of n and even values of n. Order of Growth ignoring the constant would be `n`

`Worst Case:` We always prefer worst case as a way to tell the complexity of an algorithm. For above algorithm order of
growth would be `n`.
