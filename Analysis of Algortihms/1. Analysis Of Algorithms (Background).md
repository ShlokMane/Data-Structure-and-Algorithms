# Analysis of Algorithms

Analysis of algorithm is important before we start writing code, because for a given problem there can be multiple
solutions and as a developer how are we supposed to understand which solution is better. Let's start by analysing sum of
n natural number solution.

Example:
 1. Input: n = 3
    Output: 6  (1+2+3)
 2. Input: n = 5 
    Output: 15 (1+2+3+4+5)

Let's discuss three solution for the above given problem.

```java
/***
 * This solution uses mathematical equation to solve the problem
 */
int fun1(int n) {
    return n*(n+1)/2;
}

/***
 * This solution runs a loop from i = 1 to i = n and add i to the result
 */
int fun2(int n) {
    int res =0;
    for(int i = 1;i <= n; i++) {
        res= res+i;
    }
    return res;
}

/***
 * This solution is same as above but instead of adding i to res directly we run one more loop from j=1 to i and do 
 * res++ i.e. add i to res via j
 */
int fun3(int n) {
    int res = 0;
    for(int i = 1;i <= n; i++) {
        for(int j = 1; j <= i; j++) {
            res++;
        }
    }
    return res;
}
```

All of the above solution gives us a correct output but which of them is the fastest? How do you define fastest? If you 
run a bad algorithm on a fast processing system it might run faster than good algorithm on slow processing system. Or if 
you have written code in c/c++ it might run faster than other languages. To solve this issue we have something called 
`asymptotic analysis` which is a theoretical and mathematical analysis.
