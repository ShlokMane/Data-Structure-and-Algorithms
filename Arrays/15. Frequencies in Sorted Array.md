Given a sorted array find frequencies of all the elements in it.

```
Example - 
I/P - arr[] = [10, 10, 10, 25, 30, 30]
O/P - 10 3
      25 1
      30 2  
```

Keep frequency as one and start traversing the array from index 2, if previous element is same as current element then
increase the frequency. If not equal then print the frequency of previous element and reset the freq = 1 and go to next
element.

```java
void printFreq(int[] arr, int n) {
    int freq = 1, i = 1;
    while (i < n) {
        while (i < n && arr[i] == arr[i - 1]) {
            freq++;
            i++;
        }
        System.out.println(arr[i - 1] + " " + freq);
        i++;
        freq = 1;
    }
    if (n == 1 || arr[n - 1] != arr[n - 2]) {
        System.out.println(arr[n - 1] + " " + 1);
    }
}
// Time Complexity - O(n)
```

Edge cases are if array size is 1 or if `arr[n-1]!=arr[n-2]`. First is easy to understand but the second one happens
because if last element is not equal to second last we print the frequency of second last element and then for last
element we reset `freq=1` and make i++ which will make `i=n` and the outer while loop condition is breaks. If last two
elements are equal only the inner loop will stop, and then we print the frequency.
